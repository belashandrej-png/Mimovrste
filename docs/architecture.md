## 3.2. Взаимодействие модулей

### Поток данных (ETL-процесс):

1. **Extract (Извлечение):**
   - Чтение CSV файла чанками по 50 000 строк
   - Скорость извлечения: ~53 000 строк/сек
   - Проверка целостности данных

2. **Transform (Трансформация):**
   - Очистка цен (удаление лишних разделителей)
   - Преобразование типов данных
   - Фильтрация выбросов
   - Обогащение данными (год, месяц, дата)

3. **Load (Загрузка):**
   - Сохранение в Parquet формат
   - Создание профиля обработки (JSON)
   - Периодическое сохранение (каждые 500 000 строк)

## 3.3. Технологический стек

| Компонент | Технология | Обоснование |
|-----------|------------|-------------|
| Язык программирования | Python 3.9+ | Богатая экосистема библиотек для анализа данных |
| Обработка данных | Pandas | Эффективная работа с табличными данными |
| Визуализация | Matplotlib, Seaborn | Гибкость и качество графиков |
| Хранение | Parquet (zstd) | Высокая степень сжатия, быстрый доступ |
| Числовые расчеты | NumPy | Векторизованные операции |

## 3.4. Характеристики производительности

| Метрика | Значение |
|---------|----------|
| Объем обработанных данных | 26 ГБ |
| Количество обработанных строк | 480 900 277 |
| Время обработки | ~9039 сек (~2.5 часа) |
| Скорость обработки | ~53 215 строк/сек |
| Размер выборки | 50 002 строки |
| Использование памяти | Оптимизировано через chunking |

## 3.5. Масштабируемость

### Горизонтальное масштабирование:
- Возможность распараллеливания обработки чанков
- Поддержка распределенных вычислений (при необходимости переход на Spark)

### Вертикальное масштабирование:
- Увеличение размера чанка при наличии памяти
- Оптимизация алгоритмов очистки

## 3.6. Отказоустойчивость

- **Периодическое сохранение:** Каждые 500 000 строк
- **Возобновление обработки:** Возможность продолжить с места остановки
- **Валидация данных:** Проверка корректности на каждом этапе
- **Логирование:** Фиксация прогресса и ошибок
