# 5. Обработка данных

## 5.1. Выбор технологии обработки

Для решения задач анализа данных используется подход **Пакетной обработки (Batch Processing)**.

### Инструментарий:
* **Язык:** Python 3.9+
* **Библиотека:** Pandas (для манипуляций с DataFrame)
* **Алгоритмы:** Статистический анализ, метод межквартильного размаха (IQR) для поиска выбросов, индексный анализ (метод Ласпейреса).

### Обоснование:
Хотя методичка рекомендует использование фреймворков **Apache Spark** или **Hadoop MapReduce** для распределенных вычислений, для объема выборки в 50 000 строк (после агрегации 26 ГБ) использование Pandas является более эффективным решением (overhead на запуск Spark-кластера превысит время расчета).
Однако архитектура модуля `src/data/loader.py` написана с использованием **chunking** (чтение по частям), что позволяет масштабировать решение на Spark при необходимости без изменения логики очистки.

## 5.2. Описание алгоритмов

### 1. Очистка цен (Data Cleaning)
* **Проблема:** Цены в CSV могут содержать лишние разделители (например, `1.200,50` или `1200.50`).
* **Алгоритм:**
  1. Замена запятых на точки.
  2. Удаление всех точек, кроме последней (разделитель дробной части).
  3. Приведение к типу `float`.
  4. Фильтрация: удаление цен `< 0` и аномально высоких цен (`> 10000`).

### 2. Поиск выбросов (Outlier Detection)
Используется **метод межквартильного размаха (IQR)** (раздел 5.2 методички):
* Вычисляются квартили Q1 (25%) и Q3 (75%).
* `IQR = Q3 - Q1`.
* Выбросами считаются значения за пределами `[Q1 - 1.5*IQR; Q3 + 1.5*IQR]`.

### 3. Индексный анализ (Ласпейрес)
Расчет индекса цен с фиксированным весом (структура корзины товаров базового периода).
* Формула: $I_L = \frac{\sum p_1 q_0}{\sum p_0 q_0}$

## 5.4. Оптимизация

* **Чтение чанками (Chunking):** Позволяет обрабатывать файлы, объем которых превышает объем оперативной памяти (RAM).
* **Типы данных:** Использование категориальных типов (`category`) для колонок с повторяющимися строковыми значениями (например, `category_name`) снижает потребление памяти.
