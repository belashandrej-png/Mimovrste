# 5. Обработка данных

## 5.1. Выбор технологии и подход к обработке

Согласно классификации способов обработки информации (рис. 7 методички), в проекте реализован подход **пакетной обработки (Batch Processing)**.

### Обоснование выбора технологий:

Для реализации аналитического решения использована гибридная архитектура, позволяющая демонстрировать навыки работы как с инструментами Data Science, так и с технологиями Big Data.

1. **Прототипирование (Pandas + NumPy):**
   Основной анализ (очистка, статистика, визуализация) выполнен на языке **Python** с использованием библиотеки **Pandas**. Это обосновано тем, что для выборки объемом 50 002 строки использование Pandas обеспечивает наивысшую скорость разработки и отладки алгоритмов.

2. **Архитектура распределенных вычислений (Apache Spark):**
   Исходные данные имеют объем **26 ГБ (480 млн строк)**, что превышает лимит оперативной памяти для полной загрузки. Для обработки полного объема данных применен подход **Chunking** (потоковое чтение), который концептуально аналогичен парадигме **MapReduce**.
   
   Архитектура модуля `loader.py` спроектирована так, что алгоритмы очистки и трансформации могут быть масштабированы на кластер **Apache Spark** без изменения логики. Логика обработки данных переносится к данным, а не данные к вычислениям, что является основным принципом экосистемы Hadoop/Spark.

## 5.2. Описание алгоритмов и методов анализа

### 5.2.1. Очистка и предобработка данных (ETL-процесс)

Процесс обработки реализует классический ETL (Extract, Transform, Load).

**Алгоритм очистки цен:**
Цены в исходном CSV-файле содержат лишние разделители (европейский формат). Реализована функция нормализации, которая заменяет запятые на точки и удаляет лишние разделители тысяч.

**Этапы трансформации:**
1. **Extract:** Чтение файла `mimodump-dataset.csv` чанками по 50 000 строк (аналог разделения на блоки в HDFS).
2. **Transform:** 
   - Преобразование строковых значений цен в `float`.
   - Фильтрация некорректных значений (цены < 0 или > 10 000 €).
   - Обогащение: извлечение года и месяца из поля `parsed_at`.
3. **Load:** Сохранение обработанной выборки в колончатый формат **Apache Parquet** со сжатием `zstd`, оптимизированный для аналитических запросов.

### 5.2.2. Статистические методы анализа

Для решения задач курса применены следующие методы:

| Задача | Метод | Описание |
| :--- | :--- | :--- |
| **Динамика цен** | Группировка и агрегация | Расчет средних цен по категориям за 2021-2023 гг. |
| **Сезонность** | Временной ряд | Анализ колебаний цен по месяцам (выявление пиков и спадов). |
| **ИПЦ** | Базисный индекс | Расчет индекса потребительских цен (База: Янв 2021 = 100). |
| **Волатильность** | Коэффициент вариации (CV) | Оценка стабильности цен: $CV = \frac{\sigma}{\mu} \times 100\%$ |
| **Выбросы** | IQR (Interquartile Range) | Метод Тьюки для поиска аномальных цен ($1.5 \times IQR$). |
| **Индекс Ласпейреса** | Взвешенный индекс | Расчет с учетом структуры корзины (весов) базисного периода. |

### 5.2.3. Алгоритм выявления ошибок (Поиск некорректных данных)

В соответствии с разделом 5.2 методички, реализован алгоритм автоматического выявления ошибочных данных.
1. Рассчитываются квартили $Q1$ (25%) и $Q3$ (75%).
2. Вычисляется межквартильный размах $IQR = Q3 - Q1$.
3. Границы допустимых значений: $[Q1 - 1.5 \cdot IQR; Q3 + 1.5 \cdot IQR]$.
4. Все значения за пределами границ помечаются как выбросы и исключаются из анализа.
   *Результат:* Обнаружено **5 906 выбросов (11.81%)**, что подтверждает необходимость этапа очистки.

## 5.3. Масштабирование на распределенные фреймворки (Apache Spark)

Для демонстрации компетенций в области Big Data, логика обработки данных адаптирована для выполнения в среде **Apache Spark**. Ниже приведен пример того, как локальные операции Pandas транслируются в распределенную обработку через **Spark SQL / PySpark**.

**Пример кода масштабирования (PySpark):**

```python
# Инициализация SparkSession
from pyspark.sql import SparkSession
from pyspark.sql.functions import col, regexp_replace, avg

spark = SparkSession.builder.appName("MimovrsteAnalysis").getOrCreate()

# 1. Чтение данных (аналог pd.read_csv)
df = spark.read.csv("hdfs://path/to/mimodump-dataset.csv", header=True, inferSchema=True)

# 2. Очистка цен (распределенная трансформация)
# Аналог apply(clean_price) в Pandas
df_cleaned = df.withColumn(
    "price_clean",
    regexp_replace(col("price"), ",", ".").cast("double")
)

# 3. Агрегация (аналог groupby)
# Аналог df.groupby('category')['price'].mean()
result = df_cleaned.groupBy("category_name") \
                   .agg(avg("price_clean").alias("avg_price")) \
                   .orderBy(col("avg_price").desc())

result.show()
